1. 쿠키는 브라우저에 저장되는 데이터 중 하나 입니다. 쿠키는 어떤 목적으로
사용되며 브라우저에 저장되기까지 과정을 설명하시오.
    목적 : 개인 서비스를 제공하기 위해서
    서버가 응답 헤더 Set-Cookie에 값을 지정 -> 브라우저는 쿠키 등록 -> 매
    요청마다 요청 헤더 Cookie 항목을 통해서 서버로 전송

2. 쿠키를 사용할때 발생할 수 있는 문제와 이를 보완할 수 있는 방법을
설명하시오.
문제점
    1) 요청 헤더를 통해서 전송되므로 민감한 데이터의 노출
    2) 쿠키의 데이터가 많을 수록 네트워크 부담
    보완방법
    세션을 이용해서 개인 데이터를 저장할 수 있는 공간을 서버에 마련한다.

3. 스프링 웹 MVC 프레임워크에서 쿠키 값 한개를 요청 처리 메서드에 주입 받는
방법을 설명하시오.
@CookieValue 애노테이션을 적용한다.

4. @SessionAttribute, @SessionAttributes의 용도를 설명하시오.
    1) @SessionAttribute : 세션 값 한개를 요청 메서드에서 주입 받을 때
    2) @SessionAttributes : 지정된 속성의 데이터 유지 범위를 세션범위로
    변경할 때, 세션 범위의 속성 관리가 Model로 가능

5. 컨트롤러의 실행 시점에 따라 공통 적인 처리를 하기 위한 틀을 스프링 웹 MVC
프레임워크가 제공합니다. 이 틀은 HandlerInterceptor 인터페이스 이며, 각
시점별로 실행되는 메서드 3가지가 정의되어 있습니다. 이 메서드를 작성하고
실행 시점을 설명하시오.
    boolean preHandle(...) : 컨트롤러 요청 처리 메서드 실행 전 호출
    일자 20250522 이름
    void postHandle(..., ModelAndView ..) : 컨트롤러 요청 처리 메서드 실행
    ModelAndView 반환 직후 실행
    void afterCompletion(...., Exception e) : 응답을 완료한 후 실행
